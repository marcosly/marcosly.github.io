基于Hexo的博客搭建

假设本地环境已经配置好，包括git，github账号。

一、绑定Github并提交文件

涉及到一些git操作，可以用于熟悉git维护代码仓库的相关过程。

1. 绑定github

在确认本机安装了ssh之后，生成本机的密钥和公钥：

ssh-keygen -t rsa

这里rsa是指定的密钥生成算法，敲四次回车后会生成两个文件：id_rsa和id_rsa.pub。

把公钥id_rsa.pub的内容添加到github，首先打开id_rsa.pub复制里面的内容：

cd ~/.ssh

cat id_rsa.pub

通过Ctrl + insert 复制内容后进入github主页，点击settings：

如下图，点击*SSH and GPG keys*** ，再点击*New SSH key***.

将复制的公钥内容粘贴到key内，再点击 Add SSH key.

验证是否成功，我们可以通过在 Git Bash 中输入 ssh -T git@github.com 进行检验：

第一次会出现这种情况，填 yes 就行，若出现下图中的情况，则表明绑定成功：

2. 提交文件

提交文件有两种方法：

  • 第一种：本地没有git仓库
    1. 直接将远程仓库clone到本地
    2. 将文件添加并commit到本地仓库
    3. 将本地仓库的内容push到远程仓库
  • 第二种：本地有git仓库，并且已经多次commit
    1. 建立一个本地仓库进入，init初始化
    2. 关联远程仓库
    3. 同步远程仓库到本地仓库
    4. 将文件添加到本地仓库
    5. 将本地仓库的内容push到远程仓库

第一种方法详细步骤：

首先进入Github个人主页点击一个个人项目：

点击Code复制项目地址，然后进入我们准备存储git仓库的目录，如下图：

接下来，输入 git clone https://github.com/marcosly/marcosly.github.io.git 将远程仓库克隆到本地：

在本地目录中新建一个测试文件：

从此目录进入 Git Bash，输入 git status 命令查看仓库状态：

我们刚刚新建的test文件没有被追踪到，也就是没有被提交到本地仓库。现在我们通过git add命令将文件添加到临时缓冲区，再用git commit -m ”提交信息“ 将其提交到本地仓库，如下图：

如果你是第一次提交的话，就会出现让你输入用户名和邮箱：

$ git config –global user.name”marcosly”
$ git config –global user.email”marchihjun@gmail.com“

重新commit后成功。

现在输入 git push origin master 命令，将本地仓库提交到远程仓库，origin是远程主机的名字：

刷新github中的仓库会发现我们新上传的文件：

接下来删除刚才添加的文件，在本地目录中直接删除test文件，git status 发现有文件被删除：

重新执行上述更改git命令：

git add .

git commit -m “ssss”

git push origin master

第二种方法详细步骤：

首先建立一个本地仓库demo，使用git init 命令初始化这个仓库：

输入 git remote add origin https://github.com/marcosly/marcosly.github.io.git 命令关联远程仓库，接着输入git pull origin master 命令，同步远程仓库和本地仓库。

回到本地仓库，发现已经把远程仓库的内容同步到本地仓库：

接下来的步骤就与第一种方法一样，先输入 git add 和 git commit 命令，将要提交的文件添加并提交到 demo 仓库；然后再输入 git push origin master 命令，将本地仓库修改（或者添加）的内容提交到远程仓库就完成啦！！！
